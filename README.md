Описание проекта

Это система для анализа подписочной модели. Она обрабатывает данные, считает ключевые метрики, проверяет качество данных, формирует бизнес-инсайты через агентный слой и собирает итоговый отчет.

Как запустить

1) Сначала создайте и активируйте виртуальное окружение: python -m venv venv
2) Активация окружения на windows: venv\Scripts\activate;
на Linux / Mac: source venv/bin/activate
3) Установите зависимости: pip install -r requirements.txt
4) Запуск проекта: sh run.sh <ANTHROPIC_API_KEY> (<ANTHROPIC_API_KEY> — опциональный параметр)
Если ключ не передан или не задан, система автоматически работает в локальном режиме без использования LLM.


Что создается
subscriptions.csv — синтетические данные (если их нет)
report.md — итоговый отчет
Как устроена система

Процесс состоит из нескольких шагов:

Подготовка данных (загрузка или генерация)
Расчет метрик (выручка, churn, ARPU, активные пользователи)
Проверка качества данных
Генерация аналитических выводов (агент или локальная логика)
Формирование отчета
Agentic подход

Агент используется только для интерпретации данных. Он не считает метрики, а объясняет их:

определяет тренды
находит причины изменений
формирует бизнес-выводы
дает рекомендации

Все расчеты выполняются обычным кодом, чтобы результаты были точными и воспроизводимыми.

Проверки данных

Система проверяет:

пропуски в данных
корректность user_id и month
валидность тарифов и статусов
соответствие оплат и статусов
логическую консистентность активности пользователей
сверку метрик с сырыми данными

Также есть предупреждения по высоким churn и низкому удержанию.

Почему так сделано

Метрики считаются кодом, потому что это должно быть точно и проверяемо.
Агент используется только для анализа, потому что интерпретация данных не имеет строгих правил.

Такой подход делает систему надежной и понятной для бизнеса.
